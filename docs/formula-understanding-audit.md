# 公式理解审计草稿

这份文档不是实现清单，也不是 UI 文案。它用来约束后续实现：先读懂公式在市场里回答的问题，再决定哪些公式进入计算、哪些留在研究层。

## 我刚才的问题

我之前做错的地方是：把公式名、来源 hash 和若干表达式搬进代码，误以为这就等于“理解公式体系”。

真正应该先完成的是三件事：

1. 明确每个变量在真实市场里的含义。
2. 明确每组公式回答哪个风险问题。
3. 明确上一层的输出如何成为下一层的输入。

否则 Lab 会变成“公式展柜”，不是能帮用户判断市场路径、成本、仓位和失效条件的工具。

## 统一变量语言

这些变量不能混用：

| 变量 | 含义 | 不能替代 |
| --- | --- | --- |
| `P` / `S` / `markPrice` | 当前观察价格或回放游标价格 | 不能直接当成交成本 |
| `entryPrice` | 用户准备进入或已经进入的价格 | 不能自动等于现价 |
| `costAnchor` | 由历史 OHLCV/VWAP 得到的市场成本锚 | 不是主力成本，也不是预测价 |
| `S0` / `startPrice` | LP 或 hedge 的建仓基准价 | 不能等于行权价 |
| `K` / `strikePrice` | 期权行权价 | 不能当 LP 区间中心 |
| `T` | 持仓或到期期限 | 必须说明单位，当前默认天 |
| `sigma` / `s` / `v` | 年化波动率或 IV | 历史波动与市场 IV 不能偷换 |
| `d` | GetDelta 里的目标增量/局部斜率约束 | 不能用历史收益分位数替代，若解释成收益必须回到具体仓位和费用结构 |
| `L` | AMM 流动性规模 | 不是投入本金本身 |
| `H` | 线性 hedge 规模 | 不是标准 Delta，通常来自局部斜率 |
| `F` | 手续费缓冲 | 不能当确定收益 |

## 公式家族的真实含义

### 1. 价格路径与市场成本

核心不是计算一个好看的波动率，而是先确认所有后续公式是否站在同一条路径上。

公式：

```text
r_t = ln(P_t / P_{t-1})
sigma_ann = std(r_t) * sqrt(N)
VWAP = sum(TypicalPrice_i * Volume_i) / sum(Volume_i)
Cost_vwap = sum(Close_i * Volume_i) / sum(Volume_i)
d_cost = P_last / Cost_vwap - 1
```

市场问题：

- 当前价格相对历史可观察成交成本是高估、低估，还是贴近？
- 成交量是否可靠？如果成交量不可靠，VWAP 只能降级为参考。
- 这段路径能否支持后面的 IV、LP 区间、资金费率判断？

这层输出应该是动态路径，不是固定横线。成本带必须随 K 线窗口滚动。

### 2. GetDelta 是波动带，不是期权 Delta

这组公式的价值是把 `entryPrice + T + sigma + d` 映射成价格带。这里的 `T` 是本次仓位的持仓/到期时间，不是拿历史样本校准出来的统计窗口。

公式：

```text
e_T = sqrt(T / (365 * 2pi))
r_T = ((1 + sigma * e_T) / (1 - sigma * e_T))^2
K = P * (d * r_T - d + 1)^2 / r_T
H = K * r_T
L = K / r_T
```

从 Desmos 原式看，`K` 的解法让区间内 payoff 曲线在 `x=P` 附近满足局部斜率 `g'(P)=d`。所以 `d` 先是公式里的目标增量/斜率约束；只有放回“用什么仓位赚这段增量、费用和滑点如何扣除、到期后如何处理”的结构里，才可以被解释成策略收益目标。

市场问题：

- 如果我在任意价格 `P` 进入，给定期限和波动率，价格带在哪里？
- 这组带能不能给挂买、挂卖、止损/失效条件一个初始尺度？
- `d` 不是孤立历史收益率，它必须被手续费、滑点、仓位、融资成本重新解释。

边界：

- 当 `sigma * sqrt(T/(365*2pi))` 接近 1，公式失稳。
- 它不能替代 Black-Scholes Delta。

### 3. Black-Scholes 是共同坐标系

BS 层不是为了替用户预测，而是给期权风险一个标准参照系。

公式：

```text
d1 = (ln(S/K) + (r + sigma^2/2)T) / (sigma * sqrt(T))
d2 = d1 - sigma * sqrt(T)
C = S N(d1) - K e^(-rT) N(d2)
P = K e^(-rT) N(-d2) - S N(-d1)
```

市场问题：

- 方向风险用 Delta 看。
- 曲率风险用 Gamma 看。
- 时间损耗用 Theta 看。
- 波动率暴露用 Vega 看。

重要边界：

- Pine 脚本里的高阶 Greeks 很多来自时间序列差分，不能和严格偏导混在一起。
- 加密市场里的利率、资金费率、跳跃和肥尾会破坏 BS 假设。

### 4. LP 风险来自库存曲线

LP 不是“收益率产品”，而是自动换仓的库存曲线。

Uni v3 区间内：

```text
x_real(p) = L / sqrt(p) - L / sqrt(p_upper)
y_real(p) = L * sqrt(p) - L * sqrt(p_lower)
V_lp(p) = x_real(p) * p + y_real(p)
```

Uni v2 hedge 简化：

```text
V(S) = 2L(sqrt(S) - sqrt(S0)) - H(S - S0) + F
dV/dS = L / sqrt(S) - H
H_neutral(S0) = L / sqrt(S0)
```

市场问题：

- 价格移动后 LP 实际持有什么 token？
- hedge 只是压平局部斜率，不能消灭曲率。
- v3 的风险会集中在区间边界，不能用 v2 的平滑直觉偷换。

### 5. 流动性指纹不是预测价格

Laplace、log-Laplace、covered call fit 等图不是说价格一定按某个分布走。

它们回答的问题是：如果我想让 LP 组合长得像某个 payoff 或概率密度，流动性应该怎样分布。

市场问题：

- `mu/lambda/kappa/m` 不是交易信号，而是把目标分布切成 LP 区间权重的参数。
- `m` 越大，曲线越平滑，但执行、tick、gas、管理成本都会上升。
- 这层应该服务于“构造 LP 组合”，不是服务于“预测明天涨跌”。

### 6. AMM 几何是底座，不是策略

Lambert W、liquidity cone、Numoen invariant 等图用于解释 AMM 不变量和储备关系。

公式语义：

```text
xy = k
xy = z^2
W(z) * exp(W(z)) = z
```

市场问题：

- 为什么 LP payoff 可以从库存和不变量里长出来？
- 为什么某些 AMM 曲线可以写成解析形式？
- 为什么流动性深度可以看成曲面，而不是一条孤立曲线？

边界：

- Lambert W 图不是买卖信号。
- Desmos 里的连续曲线落地到链上会遇到 tick、分支、数值稳定性和执行成本。

### 7. 资本效率只回答资金利用率

资本效率公式：

```text
CE(x, alpha) = 1 / (1 - ((1 - x) / (1 + alpha * x))^(1/4))
```

市场问题：

- 区间收窄能节省多少资本？
- 上下界偏斜是否符合方向观点和风险预算？
- 效率提升是否值得更高出界概率和再平衡频率？

边界：

- 资本效率不是收益率。
- 不能脱离波动率、成交量、手续费和出界压力讨论。

### 8. 组合价值才是最终检查层

最终不是问某个公式对不对，而是问组合路径稳不稳。

```text
Portfolio(p) = LP(p) + Option(p) - H * (p - p0) + Fees - Funding
```

市场问题：

- LP 腿、期权腿、线性 hedge、手续费、资金费率在同一条路径上合起来是否更稳？
- Delta 中性是局部状态，路径走远后必须重新计算。
- 用户关心的“能不能赚钱”应该落在路径回放和失效条件上，而不是落在公式展示上。

## 这些公式应该合并成的能力

不是把公式做成 12 张卡，而是合成 4 个能力：

1. 市场路径能力：OHLCV -> 成本锚、波动、回撤、成本带。
2. 波动定价能力：entry/T/IV/d -> 波动带、BS Greeks、均价口径。
3. 流动性库存能力：LP 区间/AMM 不变量 -> 库存、价值、效率、分布权重。
4. 组合执行能力：LP + option + hedge + fees + funding -> 回放、挂单、失效。

UI 不应该先展示公式。UI 应该先让用户回答：

1. 我看哪个品种和周期？
2. 我想用哪个入场价和持仓窗口？
3. 主要盈利来源是成本回归、手续费、波动率错配，还是资金费率？
4. 这条路径上哪里失效？

公式只应该在用户点开“为什么”时出现。

## 回测引擎缺口

当前应用里的实现只能叫“现货路径回放”：现金、现货底仓、下一根 K 线限价成交、目标/失效/到期退出。它不能证明完整公式策略有效。

完整组合回测必须先有这些领域对象：

1. `PortfolioPosition`：一个账户在某个入场日期后的现金、现货、LP、期权、perp hedge、费用和资金费率状态。
2. `Leg`：spot leg、option leg、LP segment leg、perp hedge leg，各自有入场、退出、估值和到期规则。
3. `Event`：fill、option expiry、LP rebalance、fee accrual、funding settlement、hedge adjust、liquidity split/merge。
4. `MarkToMarket`：每根 K 线只用当时已知输入重估组合价值，不能引入未来参数。
5. `RebalancePolicy`：把流动性指纹的连续密度积分成真实 segment weights，再按 tick/手续费/维护成本决定是否重整。

在这些对象缺失前，回放结果只能用于检查“成本路径 + GetDelta + 账户输入”的现货条件链，不能用于验证 LP 模拟、期权组合、流动性治理或完整仓位管理。

## 目前仍然没有读透的部分

这些不能假装已经实现：

- Lambert W 高斯轮廓和 AMM 不变量之间的严格推导关系，还需要按原图逐式重读。
- Numoen Math 需要对照协议机制，不能只靠 Desmos 图确认。
- 流动性指纹如何从连续分布离散成真实 LP 区间权重，需要写出积分和离散化规则。
- Asian/Bachelier 与 LP payoff 的贴合关系，目前只知道用途，不能直接当策略结论。
- 资金费率目前只有 Pine 里的 TWAP 近似，还没有接真实永续数据和结算制度。

这些部分进入 UI 时，只能标成“研究层”，不能参与默认挂单结论。
