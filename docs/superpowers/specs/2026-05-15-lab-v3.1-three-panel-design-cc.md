# Market Lab v3.1 · 三栏可折叠面板设计

> 状态：已确认，待落实施计划
> 范围：把 v3 的"主图占满 + FAB 滑入抽屉"改为三栏（左 tab 面板 + 中主图 + 右市场列表）
> 约束：保留所有 v3 PR-A/B/C/D 已完成的组件内容和 domain 逻辑；零数值回归

## 0. 背景

v3 完成 TradingView 风骨架（TopBar + 主图占满 + 3 FAB 抽屉）后，用户反馈"只看一个图不够"，希望：
1. 主图不占满
2. 右侧标的列表，按市场分组切换
3. 左侧策略看板可展开/收起，含决策/计算/设置 tab

本轮做**布局结构调整**，不改 domain 计算，不改 narrative，不改 PR-1 日基弱化。

## 1. 总体目标

- **D（决定）**：三栏 grid（左面板 / 主图 / 右面板），两侧可折叠到 36px 窄边条；左面板 3 tab；右面板 4 市场分组
- **N（不做）**：不改 domain 公式数值，不动 narrative，不删任何 v3 已完成的组件功能
- **R（风险）**：CSS grid 列宽切换 + lightweight-charts ResizeObserver 联动；persistedRef 状态切换不应导致 hydration 闪烁

## 4. 组件层

### 4.1 `LeftPanel.vue`（新）

**Props**：
- `open: Boolean`
- `activeTab: String` ('decision'|'compute'|'settings')
- `lab` 整体引用（避免大量 props 透传）

**Emit**：
- `toggle`
- `set-tab(name)`
- 4 个 tab 的事件透传：`set-profile / set-auto-profile / select-formula / override-tdpy / reset-tdpy / set-theme / reset-all`

**结构**：

```vue
<aside class="lp" :class="{ open, collapsed: !open }">
  <button v-if="!open" class="lp-edge" @click="$emit('toggle')">
    <span class="lp-edge-icon">▶</span>
    <span class="lp-edge-text">{{ tabLabels[activeTab] }}</span>
  </button>

  <template v-else>
    <header class="lp-head">
      <nav class="lp-tabs">
        <button :class="{active: activeTab==='decision'}" @click="$emit('set-tab', 'decision')">💡 决策</button>
        <button :class="{active: activeTab==='compute'}"  @click="$emit('set-tab', 'compute')">📊 计算</button>
        <button :class="{active: activeTab==='settings'}" @click="$emit('set-tab', 'settings')">⚙️ 设置</button>
      </nav>
      <button class="lp-collapse" @click="$emit('toggle')" title="收起">◀</button>
    </header>
    <div class="lp-body">
      <DecisionDrawer v-if="activeTab==='decision'" .../>
      <ComputeDrawer  v-if="activeTab==='compute'"  .../>
      <SettingsDrawer v-if="activeTab==='settings'" .../>
    </div>
  </template>
</aside>
```

> 3 个 *Drawer 组件复用现状不变。命名保留 "Drawer"（YAGNI）。

**样式**：宽度由 grid 控制，组件内只关心内部排版。

### 4.2 `RightPanel.vue`（新）

**Props**：
- `open: Boolean`
- `samples: Array` (合并的 marketSamples + stockIndex)
- `currentSource: Object` (lab.source，用于高亮当前选中)
- `loadingSampleId: String`

**Emit**：
- `toggle`
- `select-sample(sample)`

**分组**（用 PR-1 的 `inferTdpy`）：

```js
import { inferTdpy } from '../domain/market/tdpy.js'

const groupedSamples = computed(() => {
  const groups = { crypto: [], us: [], hk: [], cn: [] }
  for (const s of props.samples) {
    const basis = inferTdpy(s).basis
    if (basis in groups) groups[basis].push(s)
  }
  return [
    { id: 'crypto', label: '加密', icon: '🪙', items: groups.crypto },
    { id: 'us',     label: '美股', icon: '📈', items: groups.us },
    { id: 'hk',     label: '港股', icon: '🇭🇰', items: groups.hk },
    { id: 'cn',     label: 'A 股', icon: '🇨🇳', items: groups.cn },
  ].filter(g => g.items.length > 0)
})
```

**默认展开组**（computed）：当前 source 所属组优先；否则展开 `crypto`。

**过滤**：本地 input，按 `symbol` 或 `label` 不区分大小写包含匹配；过滤后空组不显示。

**结构**：

```vue
<aside class="rp" :class="{ open, collapsed: !open }">
  <button v-if="!open" class="rp-edge" @click="$emit('toggle')">
    <span class="rp-edge-icon">◀</span>
    <span class="rp-edge-text">市场</span>
  </button>

  <template v-else>
    <header class="rp-head">
      <input v-model="filter" placeholder="过滤代码/名称…" class="rp-filter" />
      <button class="rp-collapse" @click="$emit('toggle')" title="收起">▶</button>
    </header>
    <div class="rp-body">
      <p v-if="!filteredGroups.length" class="rp-empty">无匹配标的</p>
      <details
        v-for="g in filteredGroups"
        :key="g.id"
        :open="g.id === defaultExpandedGroupId"
      >
        <summary>
          <span>{{ g.icon }} {{ g.label }}</span>
          <em>{{ g.items.length }}</em>
        </summary>
        <ul>
          <li
            v-for="s in g.items"
            :key="s.id"
            :class="{ active: currentSource?.id === s.id, loading: loadingSampleId === s.id }"
            @click="$emit('select-sample', s)"
          >
            <span class="sym">{{ s.symbol }}</span>
            <span v-if="s.label !== s.symbol" class="lbl">{{ s.label }}</span>
          </li>
        </ul>
      </details>
    </div>
  </template>
</aside>
```

## 6. 错误与边界

- **resize 联动**：grid-template-columns 切换有 200ms transition；MainChart ResizeObserver 自动捕获新宽度并 `chart.resize()`，无需手动联动
- **首次加载**：`leftPanelOpen=true` / `rightPanelOpen=true` / `activeLeftTab='decision'`
- **持久化兼容**：旧 `lab.activeDrawer.v1` storage 不主动清理（不再被读，无害）；旧 `lab.leftPanelOpen.v1` 不存在则用默认 true
- **空状态**：`lab.rows.length === 0` 时主图区显示 empty-state，文案改为"右侧标的列表选品种加载（首次约 1~2 秒）"
- **窄屏 (< 900px)**：CSS media query 强制 `--left-w: 36px; --right-w: 36px;`，覆盖 store 持久值
- **过滤无结果**：RightPanel 过滤后 `filteredGroups.length === 0`，显示"无匹配标的"

## 10. YAGNI

明确不做：
- 面板可拖拉宽度（用户没要求，且需要拖手 + 持久化宽度，工程量高）
- 面板数量自定义
- LeftPanel 三个 tab 的 emoji 自定义
- RightPanel 单击预览 + 双击加载（统一单击加载）
- 收藏 / 最近查看分组（仅按市场维度）
- 三个 tab 的 *Drawer 组件 rename 为 *Tab（命名重构成本 > 收益）
