> 状态：已确认范围，待写实施计划
> 范围：在 v3.1 三栏基础上，加 ① 左右面板边框拖宽（持久化）② K 线下方加 KDJ + RSI 两个 pane（默认开）

## 0. 背景

v3.1 三栏布局上线后，用户反馈两点：
1. 280px / 240px 固定宽度不够灵活，左面板的"决策"内容文字密集时易溢出，希望能拖
2. 主图只看 K + 成本带不够，传统超买超卖（KDJ）+ 标准动量（RSI）需要快速定位入场位

本轮做**两个增强**，不动既有公式数值，不破坏 v3.1 三栏结构。

## 1. 总体目标

- **D（决定）**：
  1. 左右面板边框上各 4px 宽的隐形拖手，鼠标按下拖动调宽，双击重置默认；宽度持久化
  2. 主图加 KDJ pane（J 线变色 + KD 平均线）+ RSI pane（自定义缩放 + 阈值色），默认开
- **N（不做）**：
  - MACD pane（你的 Pine 中包含但工程量翻倍，留 v3.3）
  - 拖宽时实时显示像素数值小提示（多余）
  - 拖手单击收起整个面板（已经有 ◀/▶ 折叠按钮）
  - KDJ/RSI 输入参数 UI（公式参数固定 9/3 与 14，按 Pine 默认；要改就改 domain 文件常量）
- **R（风险）**：
  - mousemove 高频更新 CSS grid + lightweight-charts ResizeObserver 联动 → `requestAnimationFrame` 节流
  - pane 数量过多时主 K 线被压窄 → `rebalancePanes()` 重新分配 stretchFactor
  - localStorage 兼容：旧用户没有 `lab.leftPanelW.v1` 等新 key，走默认值；新增 `lab.chartOverlays.v1` 字段时，`persistedReactive` 字段级合并自动回退

### 5.1 `App.vue` 接拖手

#### Script 加 mousemove 节流逻辑

```js
import { ref } from 'vue'

let dragging = null  // 'left' | 'right' | null
let rafId = null
let pendingW = 0

function onResizerMouseDown(side, e) {
  dragging = side
  e.preventDefault()
  document.addEventListener('mousemove', onMouseMove)
  document.addEventListener('mouseup', onMouseUp)
  document.body.style.cursor = 'col-resize'
  document.body.style.userSelect = 'none'
}

function onMouseMove(e) {
  if (!dragging) return
  if (dragging === 'left') {
    pendingW = e.clientX
  } else {
    pendingW = window.innerWidth - e.clientX
  }
  if (rafId) return
  rafId = requestAnimationFrame(flushW)
}

function flushW() {
  if (dragging === 'left')  lab.setLeftPanelW(pendingW)
  if (dragging === 'right') lab.setRightPanelW(pendingW)
  rafId = null
}

function onMouseUp() {
  dragging = null
  document.removeEventListener('mousemove', onMouseMove)
  document.removeEventListener('mouseup', onMouseUp)
  document.body.style.cursor = ''
  document.body.style.userSelect = ''
  if (rafId) { cancelAnimationFrame(rafId); rafId = null }
}

function onResizerDblclick(side) {
  side === 'left' ? lab.resetLeftPanelW() : lab.resetRightPanelW()
}
```

## 6. 错误与边界

- **拖宽 clamp**：左 200-400，右 200-380；超界由 store 内 clamp 函数兜底
- **拖宽时 ResizeObserver**：mousemove → CSS 变量更新 → grid 列宽变 → MainChart `el` 尺寸变 → ResizeObserver 触发 `resize()`；用 rAF 节流避免 60fps × 5ms 累积
- **拖手不在折叠态显示**：`v-if="effectiveLeftOpen"` 把 resizer 整体撤掉
- **首次加载兼容**：旧用户没 `lab.leftPanelW.v1` → 默认 280；旧 chartOverlays 没 kdjPane / rsiPane → `persistedReactive` 字段级合并自动用 default true
- **KDJ/RSI 起步空数据**：前 n-1 根 K 算不出，filter 掉 null 不传给 series，避免 lw-charts 抛错
- **K 线品种不足 14 根**：`computeRSI` 应返回全 null 数组，series 收到空数据 OK（lw-charts setData([]) 是合法的）
- **窄屏 < 900px**：`effectiveLeftOpen` / `effectiveRightOpen` 已被强制 false → 拖手不渲染 → grid 列宽固定 36px / 36px
